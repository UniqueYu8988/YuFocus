# Learning Flow Regression Checklist

这份清单用于验证主工作区是否始终服务一条干净流程：

导入课程 -> 讲解一关 -> 小测 -> 用户回答 -> 老师判断并补全正确答案 -> 下一关

## Automated Guardrail

每次改教学主线后先运行：

```powershell
npm run check:learning-flow
npm run build:web
```

如果后端蒸馏相关文件也改过，再运行：

```powershell
python -m py_compile ..\src\distiller.py ..\src\bilibili_api.py ..\src\audio_fallback.py
```

## Manual Course Smoke Test

建议每次用两类课程各跑 3 到 5 个节点：

- 一门结构清楚的工具课，例如 Obsidian。
- 一门信息密度高的课程，例如医学课。

### 1. 打开课程

- 左侧当前课程能正常展示主线。
- 右侧标题区只显示当前节点标题和淡进度文字。
- 主聊天区没有阶段祝贺、徽章、错题提示等额外气泡。

### 2. 讲解一关

- 老师输出是聊天气泡样式。
- 讲解结构稳定包含：这关一句话、为什么重要、怎么理解、例子和抓手、小测。
- 内容不像摘要器，至少能让用户看完后复述核心点。

### 3. 开始小测

- 小测只问一题。
- 输入框可用。
- 底部只保留进度、当前状态、小测、下一节，不出现错题/回炉/徽章提示。

### 4. 回答正确

- 老师先判断结果，再钉牢正确答案。
- 本关变为已掌握。
- 下一节按钮可用。
- 不出现继续追问或要求再答一次。

### 5. 回答片面或错误

- 老师直接指出缺口。
- 老师直接给出一版正确答案。
- 本关仍然通过，可进入下一关。
- 不出现错题系统、回炉、补半步、复盘提示。

### 6. 进入下一关

- 下一节按钮进入顺序上的下一个学习节点。
- 不错误显示课程已完成。
- 新节点重新进入讲解状态。

### 7. 结课

- 最后一关完成后显示课程完成。
- 结课/阶段/徽章信息可以进入 toast 或学习档案。
- 主聊天流不插入额外庆祝气泡。
