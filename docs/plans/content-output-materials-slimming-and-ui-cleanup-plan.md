# 计划：成品库优先与 materials 瘦身

## 当前情况

`data/library` 已经保存 NotebookLM 清洗稿和 email 稿，并通过 `index.json` 记录视频、UP 主、标题、发布时间和成品路径。`data/materials` 仍保存完整原材料和中间产物，档案扫描、队列记录和删除逻辑仍部分依赖材料包内部文件。

用户希望长期只保留两份最终 Markdown 成品和少量有价值状态数据；失败视频可跳过或重跑，不需要为每条视频永久保存完整原材料。

## 目标

- 自动同步开启后可继续发现并处理新视频。
- 成品库成为打开、去重和长期管理的主要来源。
- `materials` 瘦身后仍能支持档案展示、状态追踪和必要重跑。
- UI 从“材料包操作”转为“打开成品文件”。

## 明确不做

- 不批量清理真实材料，除非脚本明确执行且通过成品存在校验。
- 不改变外部模型调用方式。
- 不新增数据库。
- 不扩大邮件发送范围。

## 相关模块与数据

- 后台发现：`desktop/electron/providers/sourceDiscoveryRuntime.ts`
- 队列执行：`desktop/electron/queue/queueExecutor.ts`
- 材料扫描：`desktop/electron/services/materialInventory.ts`
- 成品库：`desktop/electron/services/libraryExportService.ts`
- 删除：`desktop/electron/services/materialDeletion.ts`
- UI：`ArchivePaneParts.tsx`、`WorkbenchQueueParts.tsx`
- 数据：`data/materials`、`data/library/index.json`

## 分阶段步骤

### 阶段 1：排查自动同步

- 修改：先只读检查 Store 队列、暂停状态、日志和发现结果；定位没有视频进入处理链路的原因。
- 验证：用脚本覆盖 fresh / history 入队、队列上限和可领取任务判断。
- 完成标准：能解释当前为什么没有处理，并修复对应代码路径。
- 回退：只回退自动同步相关补丁，不触碰成品库和材料目录。

### 阶段 2：成品库索引接管

- 修改：材料扫描和去重读取 `data/library/index.json`；材料记录暴露清洗稿 / email 稿成品路径。
- 验证：瘦身材料仍出现在档案；已在成品库登记的视频不会被再次加入自动发现队列。
- 完成标准：档案和自动同步不再必须依赖完整材料包成品文件。
- 回退：保留成品库文件，恢复扫描时优先旧 materials 文件。

### 阶段 3：materials 瘦身与删除

- 修改：新增瘦身服务 / 脚本，只在成品库 Markdown 存在后删除中间文件；删除单条档案时同时清理材料包、成品 Markdown 和索引记录。
- 验证：临时数据脚本验证保留轻记录、删除重文件、路径不越界、索引同步更新。
- 完成标准：真实数据默认不被批量删除；功能路径可用于后续人工执行。
- 回退：Git 回退代码；已经被用户主动删除或脚本瘦身的真实中间文件需通过重新处理视频恢复。

### 阶段 4：UI 文件入口优化

- 修改：队列 / 档案列表改为清洗稿和 email 稿图标；档案增加删除图标；搜索栏右侧改为打开两个成品库文件夹。
- 验证：静态 UI 检查和桌面端预览。
- 完成标准：列表无文字操作按钮，缺 email 稿时不显示 email 图标。
- 回退：恢复旧操作按钮组件。

### 阶段 5：档案布局修正

- 修改：移除头像行与搜索栏之间多余分割线；修复 TED 分组切换后的垂直空白。
- 验证：桌面端截图 / smoke test。
- 完成标准：头像区紧凑，列表内容显示面积提升。
- 回退：恢复档案布局 CSS。

## 风险

| 风险 | 影响 | 处理方式 |
|---|---|---|
| 误删真实成品 | 用户丢失有效 Markdown | 删除前校验路径在 `data/library` 内；档案删除只删对应索引项路径 |
| 瘦身后档案消失 | 用户看不到历史视频 | 扫描逻辑读取轻记录和 `library/index.json` |
| 重复处理旧视频 | 浪费模型调用 | 自动发现去重纳入成品库索引 |
| 自动同步仍空转 | 用户以为开启失败 | 检查 Store、日志、队列可领取状态，并用脚本覆盖 |
