# 计划：桌面便携版和安装包体验优化

## 当前情况

项目已经具备 Electron 打包基础，但日常使用仍依赖源码目录和 `npm run dev`。这对开发方便，对普通使用不方便。

当前代码已经有这些有利条件：

- `desktop/package.json` 配置了 `electron-builder`、`build` 和 `build:portable`；
- `extraResources` 会把 Python 后端源码打入 `resources/backend`；
- `backendRuntime.ts` 在 packaged 模式下会把后端同步到用户数据目录再执行；
- 本地 Ollama 和 SenseVoice 已经以外部服务 / 外部目录方式接入，不必和软件本体强绑定。

当前主要风险是外部依赖和数据路径，不是 Electron 打包本身。

## 目标

让项目进入可双击运行的软件形态，同时保持外部模型按需调用。

第一版目标：

1. 生成 Windows x64 便携版 exe；
2. 双击可打开软件；
3. 启动时不加载大模型、不跑队列、不发邮件；
4. 需要模型时才调用 SenseVoice、MiMo、Ollama 或 SMTP；
5. 用户能在设置或自检入口看到运行环境是否可用。

第二版目标：

1. 便携版稳定后，再决定是否增加传统安装包；
2. 如增加安装包，只安装软件本体，不把大模型、CUDA 或秘密值打包进去。

## 明确不做

- 不把 Qwen3、SenseVoice、CUDA、Ollama 打进第一版 exe。
- 不自动安装 Python、Ollama 或模型。
- 不自动迁移真实 `data/materials`。
- 不在构建或启动时调用 MiMo、SMTP 或 B 站刷新。
- 不恢复旧 TTS、旧 Obsidian 或旧 Summary Pipeline 主线。

## 相关模块与数据

| 模块或数据 | 作用 | 本计划处理方式 |
|---|---|---|
| Electron 打包配置 | 生成 exe | 先复用现有 `build:portable` |
| Python 后端源码 | 字幕和材料生成 | 打包进 resources，运行时同步到用户数据目录 |
| Python 解释器 | 执行后端脚本 | 第一版要求本机已有，后续再评估内置 |
| Python 依赖 | B 站、yt-dlp、ffmpeg 桥接、jsonschema | 自检提示，不在第一版自动安装 |
| SenseVoice | 无字幕视频转写 | 外部目录，按需调用 |
| Ollama / Qwen3 | 本地 brief、邮件草稿、质量判断 | 外部服务，按需调用 |
| MiMo | 字幕清洗增强 | API Key 配置后按需调用 |
| SMTP | fresh 最近更新邮件 | 配置后按需调用，启动不发 |
| `data/materials` | 真实资料 | 不迁移、不删除、不改写 |
| Electron Store | 设置和秘密值 | 继续放 AppData，不写入 Git |

## 分阶段步骤

### 阶段 1：便携版构建基线

- 修改：
  - 核对现有 `build:portable` 是否能完成；
  - 如失败，只修最小打包路径或资源路径问题；
  - 不新增大依赖。
- 验证：
  - `npm run build:portable`；
  - 检查 `desktop/release` 是否生成 exe 和分享 zip。
- 完成标准：
  - 本机能得到一个可双击的 Windows x64 便携版。
- 回退：
  - 删除构建产物；
  - 回退本阶段对打包配置的代码修改。

状态：2026-06-30 已完成。`npm run build:portable` 成功生成便携 exe 和分享 zip。

### 阶段 2：打包版启动验收

- 修改：
  - 如打包版找不到资源、图标、后端脚本或数据根，做最小修复；
  - 明确打包版日志和数据根。
- 验证：
  - 双击 exe；
  - 打开核心页面；
  - 确认启动不触发下载、转写、MiMo、Ollama 或 SMTP。
- 完成标准：
  - 打包版能稳定打开和浏览现有资料。
- 回退：
  - 回退 runtime path 相关改动；
  - 删除构建产物。

状态：2026-06-30 已完成首轮 smoke test。便携 exe 可启动，远程调试端口可连接，页面标题为“视界专注”；启动日志未发现模型、转写、邮件或 distiller 调用。核心页面逐一点击仍待补充。

### 阶段 3：运行环境自检

- 修改：
  - 增加环境自检能力，至少覆盖：
    - Python 可执行文件；
    - Python 依赖；
    - 后端脚本路径；
    - SenseVoice 根目录；
    - Ollama API；
    - 本地模型名；
    - MiMo Key 是否配置；
    - B 站 Cookie 是否配置；
    - SMTP 是否配置。
  - UI 使用“可用 / 未配置 / 失败 / 不影响浏览”等清晰文案。
- 验证：
  - 在已配置环境下运行自检；
  - 临时关闭 Ollama 或清空某项设置时，自检能给出可理解提示；
  - 自检不展示秘密值。
- 完成标准：
  - 用户无需看代码，也能知道为什么某个能力不可用。
- 回退：
  - 移除自检入口和 IPC；
  - 保留原设置页。

状态：2026-06-30 已完成第一版。设置页新增“运行环境自检”，覆盖 Python、Python 依赖、内置后端脚本、本地 SenseVoice、Ollama、MiMo、B 站凭据和 SMTP 配置；检查过程不展示秘密值，不加载 SenseVoice 模型，不调用 MiMo 或 SMTP。

### 阶段 4：外部模型按需调用保护

- 修改：
  - 增加静态检查或日志检查，确认启动和普通页面浏览不调用：
    - SenseVoice；
    - Ollama；
    - MiMo；
    - SMTP。
  - 模型调用只出现在对应任务实际需要时。
- 验证：
  - 打包版启动后观察日志；
  - 静态检查保护关键调用入口。
- 完成标准：
  - 使用体验更轻，启动快，资源占用低。
- 回退：
  - 回退检查脚本或日志补丁，不影响主流程。

状态：2026-06-30 已完成第一版保护。新增 `desktop/scripts/check-packaged-environment-check.mjs`，验证自检入口已接入、SenseVoice 不提前导入、本地模型不生成文本、MiMo / SMTP 不被自检调用，且结果不包含秘密字段。

### 阶段 5：安装包评估

- 修改：
  - 便携版稳定后再评估 `nsis` 安装包；
  - 只安装软件本体；
  - 数据和秘密配置仍保留在用户目录。
- 验证：
  - 安装、启动、卸载不删除真实资料；
  - 旧便携版和安装版不会互相覆盖秘密值。
- 完成标准：
  - 明确是否值得进入安装包阶段。
- 回退：
  - 暂不发布安装包，只保留便携版。

状态：2026-06-30 已完成第一版评估。本阶段结论是暂不进入 NSIS 安装包。便携版已经满足双击启动和分享使用；传统安装器会引入卸载、用户数据保留、外部 Python / Ollama / SenseVoice 环境说明等额外风险，后续如需要再另开任务。

## 风险

| 风险 | 影响 | 处理方式 |
|---|---|---|
| 便携版体积过大 | 分享和启动体验变差 | 第一版不内置模型，只打包软件本体 |
| 用户没有 Python | 无法处理视频 | 自检明确提示，后续再评估内置 Python |
| 用户没有 Ollama 或模型 | 本地 brief 失败 | 本地消费层失败不影响 NotebookLM 主资料 |
| 打包版找错数据根 | 看不到旧材料或写错目录 | 第一版只显示并确认数据根，不自动迁移 |
| 安装器卸载误删资料 | 数据损失 | 安装包阶段必须明确不删除 `data/materials` |
| 自检泄露秘密值 | 安全风险 | 只显示“已配置 / 未配置”，不显示内容 |

## 依赖变化

第一阶段不新增依赖。

如果后续需要内置 Python 或做安装器增强，必须另开子任务评估体积、授权、维护成本和回退方式。

## 数据与备份

- 不删除、移动或重命名 `data/materials`。
- 不删除、移动或重命名 `data/registry`。
- 不公开或提交 Electron Store。
- 构建只允许写入 `desktop/dist`、`desktop/dist-electron` 和 `desktop/release`。
- 打包验收前后检查 Git diff，确认没有真实数据进入提交。

## 完成定义

本目标完成时，应同时满足：

- [x] 用户可以不打开源码命令行，直接双击启动软件；
- [x] 软件启动不立即加载外部模型；
- [x] 设置或自检页面能告诉用户哪些外部能力可用；
- [x] 字幕清洗、SenseVoice、MiMo、Ollama、SMTP 都保持按需调用；
- [x] 真实材料和秘密配置没有被打包、删除、迁移或公开；
- [x] 便携版是否进入安装包阶段有清楚结论。
