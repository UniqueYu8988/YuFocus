# 计划：注册表迁移与队列数据保护

状态：已完成（2026-06-20）

## 当前情况

开发运行时把注册表写到了项目根目录 `registry/`，而产品、架构和验收都声明标准位置是 `data/registry/`。现有 4 个 JSON 文件均可解析，每个 30 条视频且无重复 BV。清空队列当前还会删除材料包，存在明显误操作风险。

## 目标

- 统一注册表真实路径和文档路径。
- 零丢失迁移 4 个现有文件。
- 建立重复刷新、批量入队、subtitle-only 和清空队列数据保护回归。
- 用桌面端小样本确认真实行为。
- 建立包含当前稳定化成果的 Git 存档点。

## 明确不做

- 不恢复此前已丢失数据。
- 不批量制作视频。
- 不重构核心大文件。
- 不改依赖和基础设施。

## 相关模块与数据

- 视频注册表服务、来源 IPC、后台来源发现。
- 工作台队列存储、材料删除服务、队列 UI。
- `registry/`、`data/registry/`、`data/materials/` 和 Electron Store。

## 分阶段步骤

### 阶段 1：路径和迁移保护

- 修改：明确标准数据根；补充路径断言和迁移核验脚本。
- 验证：临时目录测试；4 个文件复制前后哈希与 JSON 统计一致。
- 完成标准：新路径可读写，旧文件无损进入新路径。
- 回退：保留旧目录备份，恢复路径修正前代码。

### 阶段 2：队列数据保护

- 修改：清空队列只清理非处理中队列项，不删除材料包。
- 验证：纯内存构造队列和材料记录，断言没有材料删除调用。
- 完成标准：清空后材料目录不变，处理中任务保留。
- 回退：回退本阶段代码；不需要恢复材料文件。

### 阶段 3：自动回归

- 修改：补强注册表、多次刷新、批量入队和清队列检查。
- 验证：相关 Node 检查、TypeScript、Python 语法检查。
- 完成标准：全部检查通过。
- 回退：逐项回退新增测试或实现。

### 阶段 4：桌面端真实回归

- 修改：原则上不改；发现问题只做本任务范围内最小修复。
- 验证：受控启动、连续刷新、小批量选择、至多 1 条真实 subtitle-only 样本、材料与注册表前后核验。
- 完成标准：不丢视频、不重复、不删除材料、不触发总结/邮件/TTS。
- 回退：停止软件，保留运行前快照和日志。

### 阶段 5：文档与 Git 存档

- 修改：更新当前状态、架构、产品和验收事实。
- 验证：检查 Git diff、未跟踪文件、秘密与运行数据。
- 完成标准：工作区内容明确，创建可回退的稳定提交。
- 回退：提交前不删除用户数据；提交后可用 Git 回退代码和文档。

## 风险

| 风险 | 影响 | 处理方式 |
|---|---|---|
| 注册表迁移覆盖文件 | 历史视频列表丢失 | 目标不存在才复制；复制后逐文件核验哈希和结构 |
| 清空队列误删材料 | 真实资料丢失 | 从服务层移除材料删除行为，并加纯内存回归 |
| 桌面端触发后台任务 | 外部调用或生成数据 | 运行前确认无 queued/processing；暂停后台；只做 1 条样本 |
| 大量未提交改动混杂 | 难以回退 | 分阶段检查 diff，提交前审计运行数据和秘密 |

## 依赖变化

无。

## 数据与备份

- 迁移前记录 4 个文件的名称、大小、时间和 SHA256。
- 新目录核验完成前不删除旧目录。
- `data/materials` 只读核验，不进行批量删除或移动。
- Electron Store 只读取必要的非秘密统计，不输出秘密字段。

## 完成定义

任务文件全部验收项完成，自动检查和桌面端回归通过，文档与代码一致，并创建稳定 Git 存档提交。

## 完成结果

五个阶段均已执行。迁移保留独立备份，真实数据未进入 Git；注册表刷新、批量选择、subtitle-only、真实材料生成和清空队列保护均获得当前代码或运行时证据。
