# 计划：结构化产品收束为 UP 主字幕清洗引擎

创建日期：2026-06-15
状态：阶段 1 已完成

## 目标状态

软件呈现为一个单用途工具：

```text
UP 主驱动的字幕清洗引擎，用于生成 NotebookLM 可导入资料。
```

不是内容平台，不是学习系统，不是多功能媒体应用。

## 阶段 1：断开主入口和明显旧入口

范围：

- 默认启动进入工作台。
- 侧边栏只保留主流程入口：字幕流水线、输出、流程、设置。
- 隐藏旧的“专注/档案/灵犀”主导航入口。
- 队列行移除手动制作精读稿按钮。
- 保留旧代码和数据，不删除模块。

验证：

- 静态脚本确认默认入口和导航文案。
- subtitle-only 纯内存脚本继续通过。
- TypeScript 类型检查通过。

结果：已完成。

## 阶段 2：设置页收束

范围：

- 设置页优先显示 B 站、输出目录、本地转写、MiMo 字幕清洗。
- 邮件、TTS、Summary 相关设置降级到遗留或高级区域。
- 不删除设置字段，不清空秘密值。

验证：

- 类型检查。
- 设置保存不丢字段。

结果：阶段 1 中已先完成最小隔离：设置页不再渲染邮件、TTS 和 Obsidian 旧配置块；旧字段和旧代码保留。

## 阶段 3：输出层收束

范围：

- 输出浏览只围绕 `.course_material/exports/notebooklm.md` 和清洗稿。
- 旧档案、旧学习包、精读稿只作为兼容读取，不作为主动作。
- 不迁移历史目录，不删除旧材料。

验证：

- 资料包扫描仍能找到 NotebookLM 输出。
- 打开/复制 NotebookLM 路径可用。

## 阶段 4：执行层最终隔离

范围：

- 默认队列和后台发现只允许 `pipelineMode: 'subtitle_only'`。
- legacy `full_editorial` 只保留为非主入口兼容路径。
- 邮件和 TTS 不从字幕队列触发。

验证：

- 纯内存脚本覆盖 subtitle-only、旧 item、非法 mode。
- 一条真实视频可完成到 `content_ready`。

## 暂不做

- 不拆 `distiller.py`。
- 不删除 Summary / email / TTS 源码。
- 不迁移旧数据。
- 不设计新产品功能。
- 不优化旧阅读体验。
