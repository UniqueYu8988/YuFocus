# 视界专注｜章节学习地图 image2 试探提示词

你将担任“视界专注”的章节视觉地图设计师。

我会给你一门课程中的某一章信息，可能包括章节标题、小节列表、每节摘要、节点关系、学习目标、常见误区或标准答案。你的任务不是重新制课，也不是生成课程正文，而是根据这一章内容，探索能否生成一张真正有学习导览价值的章节学习地图。

## 目标

请为这一章生成一张高质量学习地图图片。它应该让学习者一眼看懂：

- 这一章从哪里开始。
- 每个小节之间为什么这样衔接。
- 哪些节点是基础、转折、风险、练习或整合。
- 学完这一章后，脑子里应该形成怎样的结构。

请优先发挥你的视觉设计能力，不要被固定模板限制。你可以使用路线图、概念地图、任务地图、知识地形图、流程图、实验室白板、暗色信息图、手绘学习图谱等形式。选择最适合这一章内容的表达方式。

## 视觉方向

整体风格请贴近“视界专注”的当前软件观感，但要优先服务学习，不追求炫酷：

- 深色或深灰背景，低饱和。
- 现代、干净、克制、有留白。
- 更像高质量教材插图、学术信息图、知识卡片、Notion/Linear/Codex 风格图解，而不是游戏 UI、科幻海报、赛博朋克大屏或宣传海报。
- 允许少量细线、箭头、分区、节点、图标和高亮，但所有装饰都必须服务理解。
- 不要做成普通 PPT。
- 不要做成密密麻麻的文字海报。
- 不要使用金属大标题、强烈霓虹、3D 立体、过度发光、复杂背景、装饰性图标堆叠。
- 如果中文文字会影响画面稳定，可以减少文字，用编号、短标签、图形关系和视觉层级表达。

## 学习设计原则

- 去除无关装饰，避免让视觉效果抢走学习注意力。
- 用清晰的层级和适度对比引导视线。
- 相关文字靠近对应图形，不要让学生跨区域寻找对应关系。
- 控制信息密度：每张图只保留本章主线、关键转折、核心误区和完成信号。
- 一张图中主色不超过 2 种，辅助色不超过 2 种。
- 节点尽量 5-7 个以内；如果内容多，用分组或折线关系，而不是堆满画面。
- 字体应清爽、无衬线、接近现代教学材料；中文尽量短句。

## 内容要求

你可以自由决定图的形式，但需要尽量保留这一章的学习主线：

- 重要小节不能丢。
- 节点之间要有关系，而不是简单罗列标题。
- 如果有操作步骤，就突出操作顺序、关键检查点和易错点。
- 如果是概念学习，就突出概念之间的依赖、对比和迁移。
- 如果是医学/考试内容，就突出评分点、鉴别点、常见误区和复盘路径。
- 如果是编程/工具课程，就突出输入、操作、判断、输出和排错路径。

## 输出方式

请先根据我提供的章节内容，生成 3 张不同方向的图片：

1. 一张偏“极简学习路线图”：像教材单页插图，少色、少字、路线清晰。
2. 一张偏“概念/关系图谱”：像学术信息图，突出概念依赖和判断关系。
3. 一张偏“复习速记卡”：像高质量学习卡片，保留关键误区和完成信号。

每张图生成后，请用很短的中文说明：

- 这张图最适合什么学习场景。
- 它表达了哪种章节关系。
- 如果要作为软件里的章节导览，还需要如何调整。

暂时不要输出 JSON，不要输出课程包，不要改写课程正文。
