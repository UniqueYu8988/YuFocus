# 档案按 UP 主结构重构

状态：已完成（当前目标）

## 目标

把“输出”命名为“档案”，并把档案页从旧的文件列表心智，逐步改成以 UP 主为第一层、视频为第二层、资料文件和状态为第三层的信息结构。

## 范围

- 完成当前 UI 命名修正：“任务队列”改为“队列”，“输出”改为“档案”。
- 设计档案页按 UP 主组织的重构计划。
- 接入第一版只读 UP 主分组模型和档案页左侧 UP 主列表。
- 明确不直接迁移或删除 `data/materials` 真实资料。

## 不做

- 本阶段不迁移、不删除、不改写真实 `data/materials` 资料。
- 不改变材料扫描、队列执行、注册表或真实资料目录。
- 不删除旧档案、旧灵犀或旧学习包兼容数据。
- 不一次性重写视频卡片、队列状态联动和删除资料流程。

## 重构设计结论

档案页应从“文件列表”改成“UP 主资料库”：

```text
档案
├─ 左栏：UP 主列表
│  ├─ 全部
│  ├─ 固定 UP
│  ├─ 有资料的非固定来源
│  └─ 拾遗 / 本地文件
└─ 右侧：当前 UP 的视频资料
   ├─ UP 主概览
   ├─ 视频列表 / 卡片
   └─ 视频详情或阅读抽屉
```

## 推荐分阶段实施

### 阶段 1：建立只读分组模型

- 新增纯函数，把 `MaterialInventoryRecord[]` 分组成 `ArchiveCreatorGroup[]`。
- 分组优先使用 `creator` 匹配固定 UP 名称；再使用 `sourceId/sourceUrl/packageId` 兜底。
- “拾遗”收纳无法归属到明确 UP 的资料。
- 只写纯函数和检查脚本，不改 UI。

验收状态：已完成。

- 同一 UP 的资料进入同一个分组。
- 固定 UP 即使暂无资料，也能作为空分组显示。
- 不读取、不写入、不删除 `data/materials`。

### 阶段 2：改档案页布局

- 左侧改为 UP 主分组列表：头像、名称、资料数、最近更新时间。
- 右侧显示选中 UP 的视频资料列表。
- 保留“全部”和“拾遗”入口。
- 原有搜索保留，但只作用于当前分组或全部分组。

验收状态：已通过自动检查、类型检查和受控 Electron 真实材料显示验收。

- 点击 UP 后右侧只显示该 UP 的资料。
- 未选择 UP 时显示全部最近资料。
- 档案空状态可解释：未生成资料、或当前 UP 暂无资料。

### 阶段 3：视频资料卡片重构

- 每条视频显示：标题、BV、发布时间、资料状态、字数、最近更新。
- 主要操作保留：打开清洗稿、复制 NotebookLM 路径、定位文件。
- 删除资料仍保留确认，不做批量删除。

验收状态：已通过自动检查、类型检查和受控 Electron 真实材料显示验收。

- 已有 `exports/notebooklm.md` 的资料能打开和复制路径。
- 已有原始字幕但无清洗稿的资料能明确显示状态。
- 删除操作仍必须二次确认。

### 阶段 4：接入最近页和队列状态

- 在档案视频卡片上显示是否已在队列、处理中、失败或已完成。
- 最近页和档案页使用同一套“视频状态”文案。
- 不在档案页自动触发下载、转写或重新制作，只提供显式按钮。

验收：

- 同一 BV 的状态在最近页、队列页、档案页一致。
- 已有资料的视频不会被误认为“未处理”。

### 阶段 5：清理旧档案心智

- 文档中把当前主入口统一称为“档案”。
- 旧学习库 / 灵犀 / 旧档案映射仍作为兼容层，不混入新档案页主视图。
- 保留旧数据读取，但不恢复旧课程制作路线。

## 风险与保护

- 风险 1：`creator` 名称不稳定，可能导致同一个 UP 被拆成多个组。
  - 保护：优先使用固定 UP 的 `mid/name` 映射，名称只作为显示和兜底。
- 风险 2：旧资料没有明确 UP 信息。
  - 保护：进入“拾遗”，不强行猜测和迁移。
- 风险 3：档案页删除按钮涉及真实资料。
  - 保护：本轮不改删除逻辑；后续重构保持二次确认。
- 风险 4：一次性重写 `WorkspacePane.tsx` 风险高。
  - 保护：先抽纯分组函数，再替换 `ArchivePaneParts.tsx` UI。
