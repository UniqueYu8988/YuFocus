# 首页承接视频来源与 UP 订阅管理

状态：已完成

## 目标

把视频来源浏览的主要入口前移到首页：在“最近视频”下方新增一个独立的视频来源功能区，顶部用圆形 UP 主头像选择特定 UP，下方显示该 UP 的视频列表；固定 UP 主订阅改为按钮打开弹窗管理。

## 范围

- 首页新增“视频来源”区域，采用上下结构：上方圆形 UP 头像，下方视频列表。
- 首页最近视频保持全局最近视频，不被 UP 选择筛选。
- 首页新增“管理固定 UP”按钮，弹窗中读取关注来源、搜索、固定或取消固定 UP 主。
- 不自动下载、不自动转写、不自动加入队列；后续任务已允许用户手动点击按钮加入队列。
- 不删除字幕流水线中现有批量入队能力，避免主线断裂。

## 风险

- B 站关注列表读取依赖当前 SESSDATA，有失败可能；失败时要保留本地固定来源。
- 固定来源变化后，首页本地 registry 读取要同步刷新。
- UI 结构改变后，首页安全边界仍需静态检查保护。
