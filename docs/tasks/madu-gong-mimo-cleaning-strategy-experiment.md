# 任务：马督工 MiMo 定制清洗策略单样本实验

## 背景

当前正式链路已经确认：

- MiMo 适合作为 NotebookLM 主资料清洗层；
- 本地模型适合作为 brief / 邮件正文消费层；
- 但 UP 定制清洗规则尚未正式验证“适度精简无效信息”是否优于“尽量完整保留原文”。

项目负责人希望先选取马督工的一条视频，验证这条路线是否可行。当前成本不是主要约束，目标是更优秀的输出结果。

## 目标

围绕马督工单条视频，对比：

1. 当前正式清洗稿；
2. 更完整保留原文的 MiMo 清洗；
3. 适度精简降噪的马督工定制 MiMo 清洗。

判断哪种策略更适合作为 NotebookLM 主资料。

## 范围

- 只读正式 `data/materials` 中已有马督工材料包；
- 真实调用 MiMo v2.5 进行实验；
- 只写入 `data/temp/` 下的实验目录；
- 输出对比结果、关键指标和人工判断。

## 不做

- 不覆盖正式 `exports/notebooklm.md`；
- 不修改队列、自动同步或主流程代码；
- 不删除、迁移或重命名真实材料；
- 不处理本地模型 brief / 邮件生成逻辑；
- 不处理 TED / 英文 ASR 问题。

## 验收标准

- 选定一条马督工视频并说明理由；
- 至少生成“完整保留”和“适度精简”两份实验清洗稿；
- 记录压缩率、关键术语保留、结构可读性和 NotebookLM 适配判断；
- 给出是否建议后续把马督工定制清洗接入正式链路的结论。

### 结果摘要

| 版本 | 字符数 | 相对正式稿 | 关键观察 |
|---|---:|---:|---|
| 正式现有稿 | 8177 | 100% | 最稳，保留时间戳和完整话题 |
| 完整保留实验稿 | 7567 | 92.5% | 更短，但有少量识别/事实修正不稳 |
| 适度精简实验稿 | 7160 | 87.6% | 出现“广告内容，略”这类编辑性省略，不适合作为资料层 |
| 保守精简实验稿 | 7476 | 91.4% | 比适度精简安全，但仍不如正式稿稳 |

### 结论

马督工不适合直接使用较激进的“适度精简”清洗策略。  
单样本显示，清洗模型一旦被允许判断弱信息，就可能省略广告、提醒或第二主题；这对 NotebookLM 主资料层不安全。

当前建议：

- 马督工 `notebooklm.md` 继续偏完整保留；
- 不追求明显压缩；
- 如果优化，应改成“结构化完整稿”：保留时间戳、全部话题、引用、数字、地区、反例和结尾提醒，只删除口癖、重复语气词和明显识别乱码；
- 摘要和邮件正文继续交给 `brief.local.md`。
